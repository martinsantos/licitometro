Reglas de validación de fechas

  • opening_date >= publication_date (orden cronológico)
  • 2024 <= year <= 2027 (rango permitido)
  • Años de 2 dígitos: 24-27 → 2024-2027. Años >= 28 se rechazan
  • NUNCA usar datetime.utcnow() como fallback → retornar None

🚨 Regla de oro

Enrichment nunca cambia workflow_state. Las transiciones son explícitas y manuales, validadas por el usuario desde la UI o via PUT /api/licitaciones/{id}/workflow.
