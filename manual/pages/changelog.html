📝 Changelog

Historial de cambios significativos en Licitómetro. Las entradas están ordenadas de más reciente a más antigua.

v2.0 — Abril 2026

🎯 HUNTER cross-source

Nueva capacidad de búsqueda cross-source durante el enriquecimiento. Cuando hacés click en "Enriquecer", el sistema ahora:

  • Extrae identificadores del texto (decreto, expediente, licitación, resolución, CD)
  • Busca esos identificadores en todas las otras licitaciones
  • Fallback a búsqueda por keywords del título (MongoDB $text, score >= 3.0)
  • Mergea datos faltantes sin sobrescribir existentes (idempotente)

Fix crítico: Las licitaciones de COMPR.AR ya no bypasean HUNTER + nodo matching. El endpoint POST /{id}/enrich se restructuró en 4 fases secuenciales que siempre corren para TODAS las fuentes.

📄 OpenDataLoader PDF

Nueva opción de extracción de PDFs usando opendataloader-pdf (Java 21). Devuelve output estructurado con kids[] hierarchy (headings, paragraphs, tables). Detrás de feature flag USE_OPENDATALOADER_PDF con fallback automático a pypdf. El BOE sigue usando pypdf directamente porque la segmentación del Hospital Notti depende de líneas que opendataloader a veces pierde.

🚀 COMPR.AR HTTP-only (sin Selenium)

Eliminación completa de Selenium en el scraper de COMPR.AR Mendoza. Ahora usa HTTP postback directo al mismo endpoint ASP.NET. Resultado: scraping 10× más rápido (82s vs 15 min) sin perder datos. Cache de pliego URLs con TTL 7 días, ~99% hit rate.

COMPR.AR Nacional también migrado a HTTP-only con fast-fail en 503 (comprar.gob.ar actualmente bloquea datacenter IPs).

🔒 Favoritos server-side

Migración de localStorage a MongoDB. Los favoritos ahora se sincronizan con el servidor, sobreviven a cambios de navegador, limpieza de caché y reinstalaciones. Migración automática al primer login.

📖 Manual en /manual

Este manual completo. Privado (requiere login), estilo amber, sidebar con búsqueda, 14 páginas + landing. Scroll continuo via /manual/all.html que fetchea todas las páginas y las stitchea en un solo documento.

🗺️ Vigencia completa

Sistema de estados (vigente / prorrogada / vencida / archivada) con validadores Pydantic, detección automática de prórrogas, extracción de meta tags para fechas, cron diario 6am que marca vencidas.

🛡️ Backup protection

  • Docker volumes con nombres explícitos (licitometro_mongo_data)
  • Script backup-mongodb.sh cada 6 horas con rotación 7 días
  • Script deploy-prod.sh con backup automático pre-deploy
  • Deploy NUNCA usa docker compose down — solo restart

🔁 CI/CD GitHub Actions

Pipeline automatizado: push a main → build en VPS (no en Actions, costo $0) → deploy con health check + rollback. CI de lint/build en cada PR para bloquear merges con código roto.

v1.5 — Febrero 2026

💼 CotizAR (wizard de 6 pasos)

Lanzamiento del asistente de cotizaciones: ítems, propuesta, empresa, marco legal, análisis IA (Groq), oferta final. Exportación a PDF y DOCX. Inteligencia de precios basada en ~2,600 adjudicaciones históricas de ComprasApps.

🌐 Nodos semánticos

Sistema de mapas de keywords fuzzy con stemming español, matching contra title/objeto/description/organization. Auto-matching en scraping + enrichment. Digest notifications configurables (none/daily/twice_daily). Acciones: email, Telegram, tag.

Nodos iniciales: Servicios IT Última Milla (1,296 matches) y Vivero (259 matches).

🧪 Lab experimental

Quick Test con Firecrawl, Compare scrapers side-by-side, Extract con IA (Groq), PDF Parser A/B, URL resolver test, Enrichment dry-run.

📊 Dashboard de estadísticas

Vista /stats con breakdowns por fuente, estado, categoría, jurisdicción, mes, tipo de procedimiento. Export a CSV.

📅 Filtros de fecha sincronizados

Los filtros nuevasDesde (first_seen_at) y fechaDesde/fechaHasta (fecha_scraping) ahora se sincronizan automáticamente para días únicos. DailyDigestStrip, QuickPresetButton "Nuevas de hoy" y PresetSelector actúan de forma coherente.

🚨 Emergency fix: Auto-workflow-transition bug

enrichment_cron_service estaba auto-transicionando TODAS las licitaciones de descubierta → evaluando cada 30 minutos. 1,349 items se corrompieron. Fix: eliminada la lógica de auto-transición, el servicio solo enriquece datos nunca toca workflow_state. Regla grabada en piedra.

v1.0 — Enero 2026

🎉 Lanzamiento BETA

  • Backend FastAPI + MongoDB (Motor async)
  • Frontend React 18 + TypeScript con Tailwind
  • 24 scrapers activos (ComprasApps, COMPR.AR, Boletín, municipios)
  • ~3,000 licitaciones indexadas
  • Auto-clasificación por 34 rubros via keyword matching
  • FilterSidebar estilo Booking.com con faceted counts
  • Daily digest 9am por Telegram (@Licitobot) y email
  • Scraping 5 veces al día (8, 10, 12, 15, 19hs), 7 días/semana

🌐 Infraestructura

  • Docker Compose production (mongodb + backend + nginx + certbot)
  • VPS Hostinger + Cloudflare DNS
  • SSL Let's Encrypt con auto-renew
  • Nginx rate limiting (10r/s API, 3r/m auth)
  • Firewall firewalld, Docker bridge trusted zone
  • IPv6 habilitado para acceso a sitios que bloquean datacenter IPv4
  • Postfix local relay para emails sin SMTP externo

🔒 Auth

  • Contraseña compartida → bcrypt hash → JWT cookie (24h)
  • Tokens públicos de 30 días para links compartibles en notificaciones
  • Middleware en server.py protege todas las rutas /api/ excepto exempted

Roadmap
